סקירה כללית

מה המוצר עושה?

🔐

שלב 1 – כניסה ללינקדאין

הסוכן נכנס לחשבון הלינקדאין שלך אוטומטית, ושומר את פרטי ה-Session כדי לא לצטרך להתחבר שוב בפעם הבאה.

🔍

שלב 2 – חיפוש משרות

הדפדפן גולש בעצמו לעמודי חיפוש המשרות לפי מילות המפתח והמיקום שהגדרת, ושולף את הפרטים של כל משרה.

🤖

שלב 3 – ניתוח AI

כל משרה נשלחת ל-Claude AI יחד עם הקורות חיים שלך. ה-AI מעניק ציון 0–100 ומסביר בעברית למה המשרה מתאימה (או לא).

📧

שלב 4 – מייל עם התוצאות

רק המשרות שעברו את סף הציון נשלחות אליך במייל מסודר עם קישורים ישירים להגשת מועמדות. כל משרה נשלחת פעם אחת בלבד.

💡 הרעיון הבסיסי: הסוכן עושה בשבילך את כל עבודת הסריקה וסינון הרעש — אתה מקבל רק את המשרות שבאמת שוות את הזמן שלך.

גישה למידע

איך הסוכן שולף מידע מלינקדאין?

🌐 Playwright – דפדפן אמיתי

הסוכן מפעיל דפדפן Chrome אמיתי (לא בוט רגיל) שנפתח על המסך. לינקדאין רואה בדיוק אותה פעילות כמו גלישה רגילה של אדם.

⏱️ עיכובים אקראיים

בין כל פעולה יש המתנה אקראית של 1.5 עד 4 שניות — בדיוק כמו אדם שגולש לאט. זה מונע חסימה על ידי לינקדאין.

🍪 שמירת Session (Cookies)

אחרי כניסה ראשונה, ה-cookies נשמרים ב-data/session.json. בריצות הבאות הדפדפן מתחבר מיד ללא הקלדת סיסמה.

🔍 תהליך החיפוש

הדפדפן עובר לעמוד חיפוש המשרות של לינקדאין עם הפרמטרים שלך (מילות מפתח, מיקום, רמות ניסיון), גולל למטה שלוש פעמים לטעינת תוצאות, ואז פותח כל משרה כדי לשלוף את התיאור המלא שלה.

🛡️ אמצעי הגנה מפני חסימה

  • Chrome User-Agent אמיתי (גרסה 124)
  • הדפדפן גלוי על המסך (לא מוסתר)
  • דגל AutomationControlled מבוטל
  • תמיכה ב-2FA – עצירה של 60 שניות לאישור ידני
⚠️ אם לינקדאין מבקש 2FA: חלון יישאר פתוח 60 שניות לאישור ידני — פשוט אשר בדפדפן שנפתח.
בינה מלאכותית

איך ה-AI מחליט מה מתאים לך?

🧠 מה Claude AI מקבל?

לכל משרה ה-AI מקבל שני קלטים:

  • 📄 הקורות חיים שלך — מתוך הקובץ cv.txt
  • 💼 תיאור המשרה המלא — כפי שנשלף מלינקדאין

⚖️ שני קריטריוני ניקוד

  • ✅ התאמה לקורות חיים — כמה הכישורים שלך מתאימים לדרישות
  • 🌐 רלוונטיות תחום — האם התפקיד שייך לאותו תחום מקצועי, גם אם הטכנולוגיה לא מוזכרת ב-CV
לדוגמה: אם הרקע שלך בפיתוח Python — תפקיד "Machine Learning Engineer" ייצא עם ציון גבוה גם אם לא כתבת ML ב-CV.

🎯 סקאלת הציונים

90–100 התאמה מצוינת — כמעט כל הדרישות מתאימות
70–89 התאמה טובה — רוב הדרישות מתאימות
50–69 התאמה חלקית — תחום סמוך עם כישורים מועברים
0–49 התאמה גרועה — לא ישלח למייל

📝 הסיבה תמיד בעברית

ה-AI מחזיר משפט הסבר אחד בעברית לכל משרה — כך תדע בדיוק למה קיבלת את המייל הזה.

לדוגמה: "תפקיד Backend עם Python ו-REST APIs תואם במדויק לניסיון הפיתוח ולרקע הטכני של המועמד."

ערבות שליחה חד-פעמית

🗄️ מסד הנתונים (SQLite)

כל משרה שנגרדת נשמרת במסד נתונים מקומי בקובץ data/jobs.db. הטבלה כוללת:

job_id
title
company
score
emailed ✓
seen_at

🚫 לא נגרד מחדש

בתחילת כל ריצה, הסוכן טוען את כל ה-IDs של משרות שכבר נראו מה-DB. משרות שכבר נסרקו בעבר — הדפדפן פשוט מדלג עליהן.

✅ זרימת ה-One-Time Guarantee

1 משרה חדשה נגרדת → נשמרת ב-DB כ-emailed=0
2 ה-AI מנתח → ציון מספיק → נכלל במייל
3 אחרי שליחת המייל → emailed=1 ב-DB
4 ריצה הבאה: is_emailed() → TRUE → לא ישלח שוב
✅ אין כפילויות. גם אם אותה משרה תופיע שוב בחיפוש בלינקדאין — הסוכן יזהה שכבר שלחת אותה ויתעלם ממנה בשקט.

המייל שאתה מקבל

📧 מה כלול במייל?

  • כותרת בעברית עם מספר המשרות שנמצאו
  • טבלה עם: שם משרה, חברה, מיקום, ציון, הסבר
  • ציונים צבעוניים: 85+ 70–84
  • כפתור "הגש מועמדות" ישיר לעמוד המשרה

⚙️ איך נשלח?

  • Gmail SMTP SSL (פורט 465)
  • נדרש App Password — לא הסיסמה הרגילה
  • נשלח HTML + Plain Text (גיבוי לכל לקוח מייל)
  • ניתן להגדיר לשלוח גם כשאין תוצאות (send_if_empty: true)

הפעלה

3 דרכים להפעיל את הסוכן

🔍

Dry Run — בדיקה

מחפש ומדרג משרות, מדפיס את התוצאות ללוג, אבל לא שולח מייל. מושלם לבדיקה שהכל עובד לפני הפעלה ראשונה.

python main.py --dry-run
▶️

Single Run — ריצה בודדת

מריץ את הצינור המלא פעם אחת — חיפוש, ניתוח, שליחת מייל — ואז יוצא.

python main.py --once
⏰

Scheduler — אוטומטי

מריץ מיד ואז חוזר אוטומטית כל N שעות (לפי schedule_hours ב-config). הטרמינל נשאר פעיל. לחץ Ctrl+C לעצירה.

python main.py
🖥️ ממשק ניהול (Web UI): הרץ python app.py ופתח http://localhost:5000 לממשק גרפי שממנו ניתן לשלוט בכל האפשרויות, לראות לוגים בזמן אמת ולשנות הגדרות — ללא עריכת קבצים.
הגדרות

config.yaml — כל מה שאפשר לשנות

מפתח ברירת מחדל מה זה עושה? טיפ
search.keywords ["Software Engineer"] מילות חיפוש בלינקדאין — רשימה הוסף תפקידים נוספים להרחבת החיפוש
search.location "Israel" מיקום גיאוגרפי לחיפוש ניסה "Tel Aviv" לתוצאות ממוקדות יותר
search.experience_levels ["2","3","4"] רמות ניסיון: 1=סטאג', 2=Entry, 3=Associate, 4=Mid-Senior הסר רמות שלא רלוונטיות אליך
search.max_jobs_per_run 30 מקסימום משרות לגרוד בריצה אחת הגדל ל-50 לכיסוי רחב יותר
search.schedule_hours 1 כמה שעות בין ריצות בmodus scheduler 0.5 = כל 30 דק', 24 = יומי
filter.min_relevance_score 70 ציון מינימלי לשליחת משרה במייל 80+ לסינון קפדני, 60+ לכיסוי רחב
filter.cv_path "cv.txt" מיקום קובץ ה-CV שמור את הקובץ כטקסט רגיל בלבד
notifications.send_if_empty false שלח מייל גם אם לא נמצאו משרות שנה ל-true לאישור שהסוכן עובד
